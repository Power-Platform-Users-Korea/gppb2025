# Power Apps 커스텀 커넥터 만들기 (초보자 가이드)
## 소개
Power Apps의 커스텀 커넥터는 외부 데이터 소스 (예: 웹 API)에 연결하여 데이터를 가져오고 사용할 수 있게 해주는 강력한 기능입니다. 이 가이드에서는 jsonplaceholder.typicode.com/posts 라는 공개 API를 사용하여 간단한 커스텀 커넥터를 만드는 방법을 단계별로 설명합니다. 이 API는 테스트 및 학습용으로 유용한 가짜 온라인 REST API입니다.

## 이 가이드의 목표:

## 커스텀 커넥터의 기본 개념 이해하기
jsonplaceholder.typicode.com/posts API에 연결하는 커스텀 커넥터 만들기  

Power Apps 앱에서 커스텀 커넥터 사용하기   
```필수 조건```
Power Apps 계정: Power Apps를 사용하려면 Microsoft 계정이 필요합니다. Power Apps 웹사이트에서 가입하거나 로그인하세요.

## 1단계: 커스텀 커넥터 만들기 시작
Power Apps 로그인: Power Apps 웹사이트에 접속하여 로그인합니다.

데이터 메뉴 이동: 왼쪽 탐색 메뉴에서 데이터를 클릭한 다음, 커스텀 커넥터를 선택합니다.

  (이미지 추가 필요 - Power Apps 데이터 메뉴에서 커스텀 커넥터 선택 강조)

새 커스텀 커넥터 생성: + 새 커스텀 커넥터 버튼을 클릭하고 빈 템플릿에서 만들기를 선택합니다.

 (이미지 추가 필요 - 새 커스텀 커넥터 메뉴에서 빈 템플릿에서 만들기 강조)

커넥터 이름 설정: 커넥터 이름을 입력합니다. 예를 들어 "JSONPlaceholder Posts 커넥터"와 같이 알아보기 쉬운 이름을 사용하세요. 계속을 클릭합니다.

 (이미지 추가 필요 - 커넥터 이름 입력 필드 강조)

## 2단계: 일반 정보 설정
일반 탭: 커넥터 마법사의 일반 탭에서 다음 정보를 입력합니다.

설명: 커넥터에 대한 설명을 입력합니다. 예를 들어 "JSONPlaceholder API에서 게시글을 가져오는 커넥터"와 같이 작성합니다.

호스트: jsonplaceholder.typicode.com 을 입력합니다. 이것은 API의 기본 주소입니다.  
베이스 URL: 비워둡니다. 호스트에 이미 기본 URL 정보가 포함되어 있습니다. 필요한 경우 / 를 입력할 수 있습니다.  
스키마: HTTPS를 선택합니다. JSONPlaceholder API는 보안 연결을 사용합니다.
 (이미지 추가 필요 - 일반 정보 입력 폼 강조)

## 3단계: 보안 설정 (인증 없음)
보안 탭: 보안 탭으로 이동합니다.

인증 유형 선택: JSONPlaceholder API는 공개 API이므로 인증이 필요하지 않습니다. 인증 유형 드롭다운에서 인증 없음을 선택합니다.

 (이미지 추가 필요 - 보안 탭에서 인증 없음 선택 강조)

## 4단계: 정의 설정 (액션 정의)
정의 탭: 정의 탭으로 이동합니다. 여기서 커넥터가 수행할 수 있는 작업을 정의합니다.

새 액션 추가: 액션 섹션에서 + 새 액션을 클릭합니다.

 (이미지 추가 필요 - 정의 탭에서 새 액션 버튼 강조)

액션 일반 정보 입력: 새 액션의 일반 섹션에 다음 정보를 입력합니다.

요약: 액션에 대한 간단한 요약을 입력합니다. 예를 들어 "게시글 가져오기"와 같이 작성합니다.
설명: 액션에 대한 자세한 설명을 입력합니다. 예를 들어 "JSONPlaceholder에서 게시글 목록을 가져옵니다."와 같이 작성합니다.
작업 ID: 액션을 식별하는 고유한 ID를 입력합니다. 예를 들어 GetPosts 와 같이 작성합니다. (내부적으로 사용되는 ID입니다.)
 (이미지 추가 필요 - 액션 일반 정보 입력 폼 강조)

요청 정의: 요청 섹션에서 + 요청을 클릭합니다.

 (이미지 추가 필요 - 요청 섹션에서 요청 추가 버튼 강조)

요청 설정: 다음과 같이 요청 설정을 구성합니다.

HTTP 동사: GET을 선택합니다. 게시글 목록을 가져오는 것은 GET 요청입니다.
URL: /posts 를 입력합니다. 전체 URL은 https://jsonplaceholder.typicode.com/posts 로 구성됩니다.
 (이미지 추가 필요 - 요청 설정 폼에서 GET 및 /posts 입력 강조)

응답 정의: 응답 섹션에서 + 기본 응답 추가를 클릭합니다.

 (이미지 추가 필요 - 응답 섹션에서 기본 응답 추가 버튼 강조)

샘플 응답 가져오기: 응답 스키마를 정의하기 위해 API의 샘플 응답이 필요합니다. 웹 브라우저에서 https://jsonplaceholder.typicode.com/posts 에 접속하여 JSON 응답을 복사합니다.

 (이미지 추가 필요 - 브라우저에서 JSONPlaceholder 게시글 응답 화면 캡쳐)

응답 본문 붙여넣고 가져오기: 복사한 JSON 응답을 본문 필드에 붙여넣고 가져오기를 클릭합니다. Power Apps가 자동으로 응답 스키마를 생성합니다.

 (이미지 추가 필요 - 응답 본문 필드에 JSON 붙여넣고 가져오기 버튼 강조)

## 5단계: 커넥터 테스트
테스트 탭: 테스트 탭으로 이동합니다.

새 연결 만들기: + 새 연결을 클릭하여 커스텀 커넥터에 대한 연결을 생성합니다.

 (이미지 추가 필요 - 테스트 탭에서 새 연결 버튼 강조)

작업 테스트: 연결이 생성되면 정의한 GetPosts 작업을 찾습니다. 작업 테스트를 클릭합니다.

 (이미지 추가 필요 - 테스트 탭에서 GetPosts 작업 테스트 버튼 강조)

결과 확인: 테스트가 성공하면 200 OK 상태 코드와 함께 응답 본문에 게시글 데이터가 표시됩니다.

 (이미지 추가 필요 - 테스트 성공 결과 화면 강조)

## 6단계: Power Apps 앱에서 커넥터 사용하기
캔버스 앱 생성 또는 편집: Power Apps에서 새 캔버스 앱을 만들거나 기존 앱을 편집합니다.

데이터 원본 추가: 앱 편집 화면의 왼쪽 데이터 원본 패널에서 + 데이터 추가를 클릭합니다.

 (이미지 추가 필요 - Power Apps 앱 편집 화면에서 데이터 원본 추가 버튼 강조)

커스텀 커넥터 선택: 커스텀 커넥터 탭을 선택하고 방금 만든 "JSONPlaceholder Posts 커넥터"를 찾아서 선택합니다.

 (이미지 추가 필요 - 데이터 원본 추가 메뉴에서 커스텀 커넥터 탭 및 커넥터 선택 강조)

커넥터 연결: 커넥터를 데이터 원본으로 추가합니다.

앱에서 액션 사용: 이제 앱에서 GetPosts() 액션을 사용할 수 있습니다. 예를 들어, 갤러리에 게시글 데이터를 표시하려면 갤러리의 Items 속성에 다음 수식을 입력합니다.

코드 스니펫

JSONPlaceholderPostsConnector.GetPosts()
 (이미지 추가 필요 - 갤러리 Items 속성에 수식 입력 필드 강조)

이제 앱을 실행하면 JSONPlaceholder API에서 가져온 게시글 목록이 갤러리에 표시됩니다.

## 결론
축하합니다! JSONPlaceholder API를 사용하여 첫 번째 커스텀 커넥터를 성공적으로 만들었습니다. 이 가이드를 통해 커스텀 커넥터의 기본 개념과 Power Apps에서 외부 데이터를 사용하는 방법을 이해했을 것입니다. 이제 다양한 웹 API에 연결하는 더 복잡한 커스텀 커넥터를 만들어 Power Apps의 기능을 확장해 보세요.
